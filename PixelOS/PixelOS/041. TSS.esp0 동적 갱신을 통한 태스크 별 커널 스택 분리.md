# 일일 목표

- [x] TSS.esp0를 동적으로 갱신하여 각 태스크 별 커널 스택을 격리 ✅ 2026-04-02

***
# 변경 사항

## 코드

**task.c**

```C
...

#include "gdt.h"

...

// 태스크 전용 스택의 top을 반환하는 내부 함수
static uint32_t task_kernel_stack_top(const Task* t)
{
    return (uint32_t)(t->stack + t->stack_size);
}

...

// 첫 번째 태스크로 컨텍스트 스위치하는 함수
void task_start(void)
{
    ...
    
    // 현재 태스크의 전용 커널 스택 Top 주소로 TSS.esp0 갱신
    gdt_set_kernel_stack(task_kernel_stack_top(g_current_task));
	
    // 해당 태스크로 컨텍스트 스위칭
    task_switch(&g_main_esp, g_tasks[g_current].esp);
	
    idle_loop();
}

// 태스크가 CPU 점유를 양보하는 함수 (협력 스케줄링)
void task_yield(void)
{
    ...
	
    // 현재 태스크의 전용 커널 스택 Top 주소로 TSS.esp0 갱신
    gdt_set_kernel_stack(task_kernel_stack_top(g_current_task));
	
    // 해당 태스크로 컨텍스트 스위칭
    task_switch(&g_tasks[prev].esp, g_tasks[g_current].esp);
}

// 태스크 종료 함수
__attribute__((noreturn))
void task_exit(void)
{
    ...
	
    // 현재 태스크의 전용 커널 스택 Top 주소로 TSS.esp0 갱신
    gdt_set_kernel_stack(task_kernel_stack_top(g_current_task));
	
    // 해당 태스크로 컨텍스트 스위칭
    task_switch(&g_tasks[prev].esp, g_tasks[g_current].esp);
	
    idle_loop();
}
```

기존 코드에서는 `TSS.esp0`가 하나의 전역 배열 주소를 가리키고 있었습니다.
이로 인해, `Ring3` 도입 이후 **어떤 태스크가 실행 중이든 상관없이 인터럽트가 발생하면 해당 주소로 점프**하여 데이터가 덮어씌워질 수 있는 문제가 발생하였습니다.

이를 해결하기 위해 **태스크 전환이 발생하는 함수**(`task_start`, `task_yield`, `task_exit`)에서 **전환된 태스크의 전용 커널 스택 Top 주소**로 **`TSS.esp0`를 갱신**하도록 수정하여, **각 태스크가 독립된 전용 스택 공간을 사용하도록 보장**하였습니다.
