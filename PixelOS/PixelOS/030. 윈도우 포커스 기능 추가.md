**window.c**

```C
...

// 활성 상태인 창
static Window* g_active_win = 0;

...

// 주어진 창을 활성 창으로 승격
Window* wm_focus(Window* w)
{
    // 창 유효성 검사
    if (!w) return 0;
	
    // 기존에 활성 창이 존재하고 사용 중일 때
    if (g_active_win && g_active_win->in_use)
    {
        // 활성 창 테두리 색 변경 (어두운 회색)
        g_active_win->border_color = COLOR_DARK_GRAY;
        window_draw_border(g_active_win);
    }
	
    // 활성 창 전환
    g_active_win = w;
	
    // 활성 창 테두리 색 변경 (파란색)
    g_active_win->border_color = COLOR_BLUE;
    window_draw_border(g_active_win);
	
    // 가장 위 레이어로 이동
    wm_bring_to_front(w);
	
    return w;
}

// 주어진 위치의 창을 활성 창으로 승격
Window* wm_focus_at(int x, int y)
{
    return wm_focus(wm_topmost_window(x, y));
}

// 현재 활성 창에 키 입력 전달
void wm_send_key(uint8_t ascii, int pressed)
{
    if (!ascii || !pressed) return;
	
    if (!g_active_win || !g_active_win->in_use) return;
	
    window_put_char(g_active_win, (char)ascii, COLOR_BLACK);
}
```

윈도우 활성 상태(작업 포커스) 관리를 위한 정적 변수를 선언하였으며, 특정 윈도우를 **활성 윈도우**로 설정하기 위한 API를 구현하였습니다.
활성 윈도우는 **테두리를 파란색으로 그리며,** 이외의 윈도우들은 **테두리를 어두운 회색으로 그리도록 시각화**하였습니다.

새 윈도우 생성 시(`wm_create_window`) **기본적으로 활성 윈도우로 전환**되도록 수정하였습니다.

키 입력을 활성 윈도우에 전달하는 함수를 구현하였습니다. (`wm_send_key`)

# 이슈 및 버그

## 증상

 - 활성 윈도우를 전환할 때 **이전에 활성 상태였던 윈도우의 클라이언트 영역 내용이 사라지는 현상** 발생
## 재현

 - 활성 윈도우 시각화 코드에서 `window_draw_border()` 대신 `window_draw_frame()`을 사용하여 활성 윈도우 전환 수행

## 원인(추정)

 - `window_draw_frame()`에서는 테두리뿐 아니라 **배경까지 포함하여 전체 프레임을 다시 그리므로** 기존 클라이언트 영역이 유지되지 않고 배경색으로 덮어씌워지는 현상이 발생한 것으로 추정됨.

## 해결 및 우회 방법

 - 백 버퍼에 클라이언트 영역을 제외한 테두리와 제목 바를 그리는 함수를 대신 사용하여 클라이언트 영역이 유지되도록 수정하였음. (`window_draw_border()`)
