# 변경 사항

## 코드

### **boot.asm**

```Nasm
[BITS 16]               ; 16 bit 리얼 모드
[ORG 0x7C00]            ; 0x7C00부터 부트 섹터 시작

;------------------------------------------------------------------
; start 레이블
;------------------------------------------------------------------
start:
    ; 세그먼트 레지스터 초기화
    cli			            ; 초기화 중 안전을 위한 인터럽트 비활성화
    xor ax, ax		        ; ax = 0x00
    mov ds, ax		        ; ds = 0x00
    mov es, ax		        ; es = 0x00
    mov ss, ax		        ; ss = 0x00
    mov sp, 0x7C00	        ; 스택 포인터를 부트 섹터 시작점으로 설정
    sti			            ; 인터럽트 활성화

    mov si, MSG_HELLO       ; 문자열 시작 주소 저장
    call print_loop         ; 문자열 출력
    
;------------------------------------------------------------------
; 외부 코드 포함
;------------------------------------------------------------------
%include "boot/print_string.asm"

;------------------------------------------------------------------
; 데이터
;------------------------------------------------------------------
MSG_HELLO db 'Hello, PixelOS Booted from HDD!', 0x0A, 0

;------------------------------------------------------------------
; 부트섹터 패딩 및 시그니처
;------------------------------------------------------------------
times 510-($-$$) db 0   ; 부트섹터 크기를 512 byte로 맞춤 설정
dw 0xAA55               ; BIOS 부트 시그니처 : 마지막 2 byte
```

디스크의 첫 섹터(`MBR`)에 기록될 **16비트 리얼 모드 부트스트랩 코드**로, BIOS가 부팅 과정에서 이를 메모리(`0x7C00`)에 로드하여 실행합니다.
마지막 2바이트를 `0x55AA`(부트섹터 시그니처)로 설정하여 부트섹터 규격을 준수하였습니다.

### **print_string.asm**
 
```Nasm
;-------------------------------------------------------------------
; 문자열 출력 루프
;-------------------------------------------------------------------
%define CR 0x0D         ; CR(Carriage Return)
%define LF 0x0A         ; LF(Line Feed)

print_loop:
    mov ah, 0x0E        ; TTY 모드 출력

.next:
    lodsb               ; al = [ds:si++]
    or al, al           
    jz .print_done      ; null terminator → exit

    cmp al, LF          ; 줄바꿈 처리인지 확인
    jne .print_char     ; 줄바꿈 처리가 아닌 경우 문자 출력 로직으로 이동

    ; 줄바꿈 처리 로직
    mov al, CR          ; 커서 위치를 같은 줄의 맨 앞으로 이동
    int 0x10            ; BIOS Interrupt
    
    mov al, LF          ; 커서를 다음 줄로 이동
    int 0x10            ; BIOS Interrupt
    jmp .next           ; 루프

.print_char:
    int 0x10            ; BIOS Interrupt
    jmp .next           ; 출력 루프 레이블로 이동

.print_done:
    ret                 ; 출력 완료 시 리턴
```

16비트 리얼 모드에서 동작하는 **문자열 출력 루틴**으로, `DS:SI`가 가리키는 문자열을 1바이트씩 읽어 BIOS 비디오 서비스(`int 0x10`)를 통해 화면에 출력합니다.
**TTY 출력 기능**(`AH=0x0E`)을 사용하며, 출력할 문자를 `AL` 레지스터에 로드한 상태에서 `int 0x10`을 호출하여 출력합니다.
또한 개행 문자 `LF(0x0A)`를 `CR(0x0D)+LF`로 변환하여 줄바꿈이 정상적으로 동작하도록 처리하였습니다.

### **Makefile**
```Makefile
ASM = nasm

BOOT_DIR = boot
BUILD_DIR = build
DISK_DIR = disk

.PHONY: all run clean

all: $(DISK_DIR)/hddisk.img

$(BUILD_DIR)/boot.bin: $(BOOT_DIR)/boot.asm
	@echo "==> Assembling bootloader..."
	$(ASM) $(BOOT_DIR)/boot.asm -f bin -o $(BUILD_DIR)/boot.bin

$(DISK_DIR)/hddisk.img: $(BUILD_DIR)/boot.bin
	@echo "==> Creating HDD image..."
	dd if=/dev/zero of=$(DISK_DIR)/hddisk.img bs=512 count=20480
	dd if=$(BUILD_DIR)/boot.bin of=$(DISK_DIR)/hddisk.img bs=512 count=1 conv=notrunc

run:
	qemu-system-i386 -hda $(DISK_DIR)/hddisk.img

clean:
	rm -f $(BUILD_DIR)/*.bin
	rm -f $(DISK_DIR)/hddisk.img
```

여러 파일에 대한 **빌드 및 실행 환경 구성 과정을 자동화**하기 위한 Makefile입니다.
`make all` 명령으로 빌드 파일을 생성하고, 이를 디스크 이미지에 기록할 수 있습니다.
`make run` 명령으로 디스크 이미지를 사용해 QEMU에서 부팅할 수 있습니다.
`make clean` 명령으로 디스크 이미지 파일과 빌드 파일을 모두 제거할 수 있습니다.

# 이슈 및 버그

## 증상

 - QEMU에서 **"No bootable device"** 메시지가 출력되며 부팅 실패

## 재현

 - `boot.asm`에서  `times 510-($-$$) db 0` → `times 512-($-$$) db 0` 으로 변경 후 빌드 및 실행

## 원인(추정)

 - 부트섹터는 **총 512바이트**이며, **마지막 2바이트**에 **부트 시그니처 0x55AA**가 위치해야 한다.
 - `times 512-($-$$)`로 패딩하면 **MBR의 512바이트를 모두 패딩으로 채운 후** 부트 시그니처가 다음 영역으로 밀리게 되어 부트 시그니처 위치 규약을 위반하게 된다.

## 해결 및 우회 방법

 - MBR의 510바이트까지만 패딩한 뒤, 2바이트 시그니처를 마지막에 기록하는 방식으로 코드를 수정하여 해결하였다.
