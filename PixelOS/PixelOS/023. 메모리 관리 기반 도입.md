# 변경 사항

## 코드

**linker.ld**

```ld
	...
    _kernel_end = .;    # 링커가 계산하는 커널 끝 주소
```

메모리 관리를 위해 **커널 이미지의 끝 주소**를 **링커가 계산하여 기록**하도록 링커 파일에 링커 심볼을 추가하였습니다.


**heap.h**

```C
#pragma once
#include <stdint.h>
#include <stddef.h>

// 힙 범위를 설정하는 초기화 함수
void heap_init(void);

// 주소 범위를 예약하는 함수
void* kmalloc_aligned(size_t size, size_t align);

// 16바이트 단위로 정렬하여 할당하는 API (kmalloc_aligned 함수 이용)
void* kmalloc(size_t size);
```

**힙 관리**를 위한 인터페이스를 선언한 헤더 파일입니다.


**heap.c**

```C
#include "heap.h"

// 힙 크기는 1MB로 설정
#define HEAP_SIZE (1024 * 1024)

// 링커 스크립트에서 제공하는 심볼
extern uint8_t _kernel_end;

// 내부 힙을 관리하는 포인터
static uint8_t* g_heap_start = 0;
static uint8_t* g_heap_end = 0;
static uint8_t* g_heap_curr = 0;

// v를 align의 배수로 올림(ceil)하여 정렬된 주소를 생성하는 인라인 함수
static inline uintptr_t align_up(uintptr_t v, size_t align)
{
    return (v + (align - 1)) & ~(uintptr_t)(align - 1);
}

// 힙 범위를 설정하는 초기화 함수
void heap_init(void)
{
    // 커널 끝 주소 다음부터 힙 영역 시작
    uintptr_t start = (uintptr_t)&_kernel_end;
    // 힙 시작 주소를 16바이트 단위로 맞춤
    start = align_up(start, 16);
	
    // 힙 관리 포인터 초기화
    g_heap_start = (uint8_t*)start;
    g_heap_curr = (uint8_t*)start;
    g_heap_end = (uint8_t*)(start + HEAP_SIZE);
}

// 2의 거듭제곱인지 판별하는 인라인 함수
static inline int is_pow2(size_t x) {
    return x && ((x & (x - 1)) == 0);
}

// 주소 범위를 예약하는 함수
void* kmalloc_aligned(size_t size, size_t align)
{
    // 0바이트 할당의 경우 무시
    if (size == 0)
    {
        return 0;
    }
    
    // 정렬 값이 0이거나 2의 거듭제곱 꼴이 아닌 경우 16으로 기본 세팅
    if (align == 0 || !is_pow2(align))
    {
        align = 16;
    }
	
    uintptr_t curr = (uintptr_t)g_heap_curr;
    uintptr_t aligned = align_up(curr, align);
	
    uintptr_t next = aligned + size;
    // 힙 범위를 넘어서는 경우(메모리 부족) 실패
    if (next > (uintptr_t)g_heap_end)
    {
        return 0;
    }
	
    g_heap_curr = (uint8_t*)next;
	
    return (void*)aligned;
}

// 16바이트 단위로 정렬하여 할당하는 API
void* kmalloc(size_t size)
{
    return kmalloc_aligned(size, 16);
}
```

**힙 영역을 커널 끝 주소부터 시작하도록 설정**하였으며, 링커 심볼(`_kernel_end`)의 주소를 기준으로 16바이트 정렬을 적용한 뒤 `HEAP_SIZE(1MB)` 만큼의 범위를 힙으로 사용하도록 초기화하였습니다.

![[메모리 구조.png]]

또한 `kmalloc()`/`kmalloc_aligned()`는 현재 힙 위치(`g_heap_curr`)에서 **정렬 조건을 만족하는 다음 주소**를 계산하여 **해당 주소를 반환**하고, 사용한 크기만큼 힙 위치를 앞으로 이동시키는 `Bump Pointer Allocation` 방식으로 동작합니다.
만약 이동 결과가 힙의 범위를 넘어가면 **메모리 부족**으로 `0`을 반환합니다.
