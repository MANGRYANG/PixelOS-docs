# 변경 사항

## 코드

**kernel_entry.asm**

```Nasm
[BITS 32]

global _start

extern kernel_main

_start:
    mov ax, DATA_SEG
    mov ds, ax
    mov es, ax
    mov fs, ax
    mov gs, ax
    mov ss, ax

    call kernel_main

    jmp $

DATA_SEG equ 0x10
```

**커널의 진입점을 설정하는 코드**입니다.
보호 모드에서 사용할 세그먼트 레지스터들을 **데이터 세그먼트 선택자(`0x10`)**로 설정한 뒤, C로 작성된 커널 메인 함수 `kernel_main()`을 호출합니다.


**kernel.c**

```C
#include <stdint.h>

void kernel_main(void) {
    // VGA 그래픽 메모리 시작
    uint8_t* vga = (uint8_t*)0xA0000;

    for(int i = 0; i < 320*200; ++i)
    {
        vga[i] = 0x0F;  // 흰색 픽셀
    }
}
```

**커널의 메인 로직을 담당하는 C 코드**입니다.
VGA 그래픽 메모리 주소인 `0xA0000`에 픽셀 값을 저장하여 **320×200 화면**을 흰색으로 채우도록 작성하였습니다.


**linker.ld**
```
ENTRY(_start)           # 커널 실행 진입점 심볼 설정

SECTIONS {              # kernel.bin의 메모리 배치 규칙 정의
    . = 0x00100000;     # Location counter = 0x00100000 (커널 시작 주소)

    .text : ALIGN(4096)
    {
        *(.text*)       # 모든 실행 코드를 하나로 묶어 배치
    }

    .rodata : ALIGN(4096)
    {
        *(.rodata*)     # 모든 읽기 전용 데이터를 하나로 묶어 배치
    }

    .data : ALIGN(4096)
    {
        *(.data*)       # 초기값이 존재하는 전역, 정적 변수를 하나로 묶어 배치
    }

    .bss : ALIGN(4096)
    {
        *(COMMON)       # 모든 COMMON 심볼들을 bss 블록에 배치
        *(.bss*)        # 초기값이 없는 전역 및 정적 변수를 하나로 묶어 배치
    }
}
```

**링커에 전달되는 링커 스크립트**로, **커널 바이너리의 메모리 배치 규칙을 정의**하는 코드입니다.

커널 실행 진입점 심볼을 `_start`로 설정하고, 커널이 실행될 시작 주소를 `0x00100000`으로 고정하였습니다.
이후 `.text`, `.rodata`, `.data`, `.bss` 섹션을 페이지 크기 경계에 맞추어 배치합니다.


**load_kernel.asm**

```
...

;------------------------------------------------------------------
; DAP 구조체
;------------------------------------------------------------------
kernel_dap:
    db 0x10                 ; DAP 구조체 크기 : 16바이트
    db 0                    ; 예약된 공간
    dw 100                  ; 읽을 섹터 수
    dw 0x8000               ; BX : 오프셋
    dw 0                    ; ES : 세그먼트
    dd 1                    ; LBA 하위 32비트
    dd 0                    ; LBA 상위 32비트

...
   
;------------------------------------------------------------------
; PM 작업 코드
;------------------------------------------------------------------
[BITS 32]

PROTECTEDMODE:
    mov ax, DATA_SEG
    mov ds, ax
    mov es, ax
    mov fs, ax
    mov gs, ax
    mov ss, ax
    mov esp, 0x0009FC00

    ; 커널 재배치
    mov esi, 0x00008000     ; source (0x8000)
    mov edi, 0x00100000     ; destination (0x100000)
    mov ecx, 12800          ; 100 * 512 bytes / 4 bytes = 12800
    cld
    rep movsd               ; 4바이트씩 복사

    jmp CODE_SEG:0x00100000 ; 커널 진입 
    
```

커널의 크기가 증가함에 따라, **DAP**에서 설정하는 `dw`**(읽을 섹터 수)**를 기존 4에서 100으로 상향하였습니다. **(2048바이트 → 51200바이트)**

이에 커널 재배치 시 복사하는 범위도 12800워드로 상향 조정하였습니다. **(12800워드 = 51200바이트)**

# 이슈 및 버그

## 증상

 - 보호 모드 진입 후 트리플 폴트(Triple Fault)가 발생하며 부팅 실패

## 재현

 - `load_kernel.asm`에서 DAP의 `dw`를 **4로 설정**하여 커널을 로드

## 원인(추정)

 - 커널 크기가 증가하면서 **4섹터(2KB)**로는 커널 이미지 전체를 메모리에 로드하지 못함
 - 그 결과 커널이 **잘린 상태로 `0x00100000`에 복사**되어 트리플 폴트가 발생한 것으로 추정됨

## 해결 및 우회 방법

 - 부트로더가 BIOS 확장 디스크 읽기로 요청하는 섹터 수를 **4 → 100으로 상향**하여 우회적으로 해결하였음
