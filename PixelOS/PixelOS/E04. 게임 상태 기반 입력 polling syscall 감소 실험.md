# 실험 보고서: [게임 상태 기반 입력 polling syscall 감소 실험]

## 실험 개요

- 실험 번호 : 04
- 관련 프로젝트 : `PixelOS`
- 실험 대상 모듈 : `app/pong.c`

***

# 서론
## 실험 목적

> Pong 애플리케이션의 입력 처리가 `SYS_KEY_DOWN` syscall을 통해 이루어진다는 점에 주목하여, 현재 게임 상태에서 실제로 필요한 키만 polling하도록 변경했을 때 입력 관련 syscall 호출량이 감소하는지 확인한다.

## 실험 범위

### 포함 범위

- `SYS_KEY_DOWN` 호출 수 계측
- `K/P`를 이용한 frame당 key polling 수 비교
- 일반 플레이, Paused, Game over 상태 별 입력 polling 구조 비교
- 입력 polling 최적화 전후의 게임 동작 유지 여부 확인

### 제외 범위

- PS/2 입력 계층 수정
- 이벤트 기반 입력 시스템 구현
- keyboard interrupt handler 수정
- input queue 구조 변경
- 마우스 입력 처리 수정
- 입력 디바운싱 구조 개선

# 본론
## 실험 가설

> 현재 Pong 입력 처리 구조에서는 이미 Paused 및 Game over 상태에서 paddle 입력을 처리하지 않도록 일부 상태 기반 분기가 적용되어 있다.
> 
> 다만 Game over 상태에서도 실제로는 사용되지 않는 `SPACE` key polling이 남아 있으므로, 이를 제거하면 Game Over 상태에서 frame당 `SYS_KEY_DOWN` 호출 수인 `K/P`가 감소할 것이다.

| 항목    | 내용                                           |
| ----- | -------------------------------------------- |
| 독립 변수 | Game Over 상태에서 `SPACE` key polling 수행 여부     |
| 종속 변수 | `K`(`SYS_KEY_DOWN`), `K/P`                   |
| 통제 변수 | `PONG_FRAME_TICKS`, 창 크기, 공 속도, 입력 조건, 측정 시간 |
| 예상 결과 | Game over 상태에서 `K/P`가 기존 2에서 1 수준으로 감소한다.    |

## 기존 구현에서의 문제점  
  
>현재 코드의 입력 처리 구조는 모든 상태에서 모든 키를 polling하는 구조가 아니다.
>기존 구현에서도 `pong_handle_paddle_input()`에서 pause 또는 game over 상태일 경우 paddle 조작 관련 키 polling 과정을 생략하고 있다.
>
> 그러나 Game over 상태에서 사용되지 않는 `SPACE` 키가 지속적으로 polling되는 구조적 문제점이 남아 있어, 본 실험에서 이를 해결하면서 성능 변화를 확인할 것이다.

```C
// pong 게임의 전역 입력 관리를 위한 내부 함수 (일시정지, 재시작)
__attribute__((section(".usertext")))
static void pong_handle_global_input(PongState* game)
{
    int restart_down = app_key_down(APP_KEY_RESTART);
    // 게임 종료 시 R 키를 누르는 순간 한 번만 재시작
    if (game->game_over && restart_down && !game->restart_was_down)
    {
        pong_init(game);
        return;
    }
  
    game->restart_was_down = restart_down;
  
    int space_down = app_key_down(APP_KEY_SPACE);
  
    // SPACE 키를 누르는 순간 한 번만 일시정지 상태 토글
    if (!game->game_over && space_down && !game->space_was_down)
    {
        game->paused = !game->paused;
    }
  
    // space_was_down 갱신
    game->space_was_down = space_down;
}
```

- `int space_down = app_key_down(APP_KEY_SPACE);` 구문에서,
  Game over 상태인 경우에도 key polling이 수행된다.

***
  
## 적용할 최적화 기법  
  
> 적용할 최적화 기법은 **게임 상태 기반 입력 polling 조정**이다.
> 
> Game over 상태에서 `SPACE` polling을 제거하여 `SYS_KEY_DOWN` 호출을 감소시킨다.

## 구현 방법  
   
| 변경 위치        | 변경 내용                                   |
| ------------ | --------------------------------------- |
| `app/pong.c` | Game Over 상태에서 Space polling을 하지 않도록 조정 |

```C
// pong 게임의 전역 입력 관리를 위한 내부 함수 (일시정지, 재시작)
__attribute__((section(".usertext")))
static void pong_handle_global_input(PongState* game)
{
    int restart_down = app_key_down(APP_KEY_RESTART);
    // 게임 종료 시 R 키를 누르는 순간 한 번만 재시작
    if (game->game_over && restart_down && !game->restart_was_down)
    {
        pong_init(game);
        return;
    }
  
    game->restart_was_down = restart_down;
  
    if(!game->game_over)
    {
        int space_down = app_key_down(APP_KEY_SPACE);
  
        // SPACE 키를 누르는 순간 한 번만 일시정지 상태 토글
        if (space_down && !game->space_was_down)
        {
            game->paused = !game->paused;
        }
  
        // space_was_down 갱신
        game->space_was_down = space_down;
    }
}
```

# 분석

## 결과 해석

> 기존 구조에서 `!game->game_over` 조건으로 인해 Game over 상태에서는 `SPACE` 키 토글이 발생하지 않으므로, Game Over 상태의 `SPACE` polling은 불필요한 syscall이다.
> 
> 최적화 이후에는 Game Over 상태에서 `RESTART` 키만 polling하도록 변경하였다.
> 이를 통해 기존에 호출되던 `SPACE` 키에 대한 `SYS_KEY_DOWN` 호출이 제거되었으며,
> 결과적으로 시스템 콜 횟수가 줄어들어 자원 효율성이 증가하였다.

# 결론

## 실험 결론

> Game Over 상태에서 불필요한 `SPACE` 키 폴링을 제거한 결과, 예상대로 프레임당 입력 시스템 콜 수(`K/P`)가 감소하였다.
> 
> 이를 통해 **게임 상태에 따른 선별적인 입력 폴링**이 시스템 자원 관리 측면에서 유의미한 효율 개선을 가져올 수 있음을 확인하였다.

## Overview

> 이번 실험에서는 **게임 상태(State)에 따른 입력 폴링 최적화가 전체 시스템 자원 효율성에 미치는 영향을 검증**하였다.
> 
> 기존 구현에서도 상태별 선별적 폴링을 수행했으나, **Game over와 같은 특정 시점에서 유효하지 않은 로직이 여전히 자원을 점유하고 있다는 점**을 발견하였다.
> 이를 개선하여 불필요한 폴링 경로를 구조적으로 차단함으로써, 매 루프마다 반복되던 무의미한 연산 비용을 완전히 제거할 수 있었다.
> 
> 이번 실험을 통해 **게임 상태를 기준으로 입력 처리 범위를 명확히 정의하면, 기능적인 부작용 없이 입력 관련 syscall 비용을 줄일 수 있음**을 확인할 수 있었다.
