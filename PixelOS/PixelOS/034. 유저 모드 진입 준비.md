## 코드

해당 섹션의 시작과 끝 주소를 사용할 수 있도록 심볼로 남겨두었습니다.

**paging.c**

```C
...

// 주소를 4KiB 경계로 올림 정렬하는 내부 헬퍼
static inline uint32_t align_up_4k(uint32_t x)
{
    return (x + 0xFFFu) & ~0xFFFu;
}

...

// 단일 4KiB 페이지 매핑을 위한 함수
int map_page(uint32_t virt, uint32_t phys, uint32_t flags)
{
	...
	
    // U/S 플래그를 부여한 경우 PDE의 U/S 플래그도 세팅
    if (flags & P_US)
    {
        g_page_directory[pde_index(virt)] |= P_US;
    }
	
    ...
	
    // 페이징이 활성화되어 있는 경우 TLB 갱신
    if (read_cr0() & 0x80000000u)
    {
        write_cr3(paging_get_directory_phys());
    }
	
    return 0;
}

// 이미 매핑된 페이지 범위에 플래그를 부여하는 함수
int paging_set_range_flags(
	uint32_t virt,
	uint32_t size,
	uint32_t flags
)
{
    // 페이지 크기가 0인 경우 무시
    if (size == 0)
    {
        return 0;
    }
	
    // 전달된 범위가 걸쳐 있는 페이지를 모두 포함시키기 위해 정렬
    uint32_t start = align_down_4k(virt);
    uint32_t end = align_up_4k(virt + size);
	
    for (uint32_t addr = start; addr < end; addr += PAGE_SIZE)
    {
        // 가상 주소에 해당하는 페이지 테이블이 존재하는지 검사
        uint32_t* pt = get_pt_for(addr);
        if (!pt)
        {
            return -1;
        }
		
        // 페이지가 물리 메모리에 실제로 연결되어 있는지 확인
        uint32_t idx = pte_index(addr);
        if ((pt[idx] & P_P) == 0)
        {
            return -1;
        }
		
        // 플래그 부여 (R/W, U/S)
        pt[idx] |= (flags & (P_RW | P_US));
		
        // U/S 플래그를 부여한 경우 PDE의 U/S 플래그도 세팅
        if (flags & P_US)
        {
            g_page_directory[pde_index(addr)] |= P_US;
        }
    }
	
    // 페이징이 활성화되어 있는 경우 TLB 갱신
    if (read_cr0() & 0x80000000u)
    {
        write_cr3(paging_get_directory_phys());
    }
	
    return 0;
}

...
```

`map_page`에 인자로 전달된 플래그에 `U/S 비트`가 지정되어 있는 경우, **해당 PTE와 함께 상위 PDE 에도 `P_US` 비트를 설정하는 로직을 추가**하였습니다.

추가로 **페이징이 활성화된 상태인 경우 페이지 엔트리를 변경한 뒤 `CR3`를 다시 기록하여 TLB를 갱신**하는 로직도 추가하였습니다.

**이미 매핑되어 있는 가상 주소 범위의 페이지 테이블 엔트리(PTE)에 권한 플래그를 추가로 부여(OR)하는 함수**인 `paging_set_range_flags`를 구현하였습니다.

인자로 전달된 시작 가상 주소(`virt`)와 크기(`size`)가 가리키는 범위를 포함하도록, 해당 범위에 걸치는 모든 페이지를 선택한 뒤, 각 페이지의 PTE에 지정된 플래그를 **OR 방식**으로 추가하는 작업을 수행하도록 작성하였습니다.

_※ OR 방식은 기존 비트는 유지하고, 지정한 비트만 1로 켜는 방식을 의미합니다._


**kernel.c**

```C

extern uint8_t __user_text_start;
extern uint8_t __user_text_end;

...

__attribute__((aligned(4096))) static uint8_t g_user_test_stack[4096];

...

// 유저 모드에서 실행할 임시 함수 (usertext 섹션)
__attribute__((section(".usertext"), noreturn))
void user_test_main(void)
{
    for (;;)
    {
        __asm__ volatile ("hlt");
    }
}

...
void kernel_main(void)
{
    ...
    
    // usertext 섹션에 U/S 플래그 설정
    if (paging_set_range_flags(
    (uint32_t)&__user_text_start,
    (uint32_t)(&__user_text_end - &__user_text_start),
    P_US
    ) < 0)
    {

            __asm__ volatile ("hlt");

    (uint32_t)g_user_test_stack,
    4096,
    P_US | P_RW
    ) < 0)
    {
        window_put_string(g_testwin,
        "[FAIL] user stack mapping failed\n",
        COLOR_LIGHT_RED);
        compositor_compose();
		
        for (;;)
        {
            __asm__ volatile ("hlt");
        }
    }

```

링커가 제공하는 **`.usertext` 경계 심볼을 참조할 수 있도록 추가**하였습니다.

유저 모드 테스트용 스택(`g_user_test_stack`)을 새로 추가하였습니다.
**시작 주소가 `4096 Byte` 경계에 맞도록 정렬**하였으며, **페이지 1개 크기(`4096 Byte`)만큼의 스택 메모리 공간을 확보**하였습니다.

`.usertext` 섹션에 들어갈 테스트용 유저 함수(`user_test_main()`)를 추가하였습니다.

**`.usertext` 섹션 범위 전체**에 대해 `U/S 비트` 를 1로 설정하여 **유저 모드(`CPL 3`)에서 접근 가능**하도록 변경하였습니다.

`g_user_test_stack`에 대해 `U/S 비트`와 `R/W 비트`를 1로 설정하여 **유저 모드(`CPL 3`)에서 접근 및 쓰기 가능**하도록 변경하였습니다.
