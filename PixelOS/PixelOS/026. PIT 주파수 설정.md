# 변경 사항

## 코드

**timer.h**

```C
#pragma once
#include <stdint.h>

// PIT 채널 0을 원하는 주파수(Hz)로 설정하는 함수
void pit_set_frequency(uint32_t hz);
```

**timer.c**

```C
#include "timer.h"
#include "io.h"

// PIT 채널 0의 입력 클럭
#define PIT_BASE_HZ 1193182

// PIT 채널 0을 원하는 주파수(Hz)로 설정하는 함수
void pit_set_frequency(uint32_t hz)
{
    // 하한 틱 : 1193182 / 65535 (16bit) = 약 18.2 Hz
    // 주파수가 너무 낮으면 기본 틱 수준으로 설정
    if (hz < 19) { hz = 19; }
    // 주파수가 너무 높으면 인터럽트 과부하 발생 가능하므로 제한
    if (hz > 1000) { hz = 1000; }
    
    // divisor 범위 : 16bit (1 ~ 0xFFFF)
    uint32_t divisor = PIT_BASE_HZ / hz;
    if (divisor < 1) { divisor = 1; }
    if (divisor > 0xFFFF) { divisor = 0xFFFF; }

    // Select channel : 00 = Channel 0
    // Access mode : 11 = lobyte/hibyte
    // Operating mode : 011 = Mode 3 (square wave generator)
    // BCD/Binary mode : 0 = 16-bit binary
    // 0011 0110 -> 0x36
    // https://wiki.osdev.org/Programmable_Interval_Timer 참고
    outb(0x43, 0x36);
    outb(0x40, (uint8_t)(divisor & 0xFF));
    outb(0x40, (uint8_t)((divisor >> 8) & 0xFF));
}
```

기존 코드에서는 타이머 인터럽트 주기가 기본 설정값에 의존하였으나, 이를 **명시적으로 제어**하기 위하여 **PIT 채널 0의 주파수를 설정하는 함수(`pit_set_frequency`)를 추가**하였습니다.


**kernel.c**

```C
...

// 유휴 상태를 위한 태스크
static void idle_task(void* arg)
{
    (void)arg;
    for (;;) {
        asm volatile("sti; hlt");   // 인터럽트로만 깨어남
        task_yield();               // 깨어나면 다른 태스크에게 CPU 양보
    }
}

// UI 작업을 위한 태스크
static void ui_task(void* arg)
{
    (void)arg;

    uint32_t last_tick = g_timer_ticks;

    for (;;) {
        // 프레임 경계 대기
        while (g_timer_ticks == last_tick) {
            task_yield();
            continue;
        }
        
        ...
        
    }
}

...

void kernel_main(void)
{   
    ...
	
    // PIT 주파수 설정 (100Hz)
    pit_set_frequency(100);
	
	...
	
	task_create(idle_task, 0, 8192);
    task_create(ui_task, 0, 8192);
    task_create(app_task, 0, 8192);
    
	task_start();
}
```

유휴 상태를 위한 태스크(`idle_task`)를 추가하고, **PIT 주파수를 100Hz로 설정**하도록 수정하였습니다.

`hlt` 명령은 유휴 상태에서만 CPU를 정지시키기 위해 **`idle_task`에서만 사용**하도록 변경하였으며, UI 태스크는 `hlt`를 통해 프레임 동기화를 수행하던 기존 방식 대신 **작업 수행 후 `task_yield()`로 CPU를 양보**하는 방식으로 변경하였습니다.
