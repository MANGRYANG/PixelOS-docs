# 변경 사항

## 코드

**graphics.h**

```C
...

// 백 버퍼의 내용을 VGA 메모리로 출력하는 함수
void gfx_present(void);

// VSYNC 동기화 함수
void gfx_wait_vsync(void);
```

기존에 VGA에 직접 그리는 예비 함수(`gfx_putpixel_front`, `gfx_fill_rect_front`)를 제거하였습니다.

기존에 **프레임 백 버퍼와 커서를 함께 오버레이하여 VGA에 출력**하는 작업을 수행하는 함수인 `gfx_present_with_cursor` 함수를 조정해 `gfx_present` 함수로 변경하였습니다.

**화면 티어링 현상을 해결**하기 위해 **VSYNC(Vertical Synchronization) 동기화 함수** `gfx_wait_vsync`를 선언하였습니다.


**graphics.c**

```C
...

// 백 버퍼의 내용을 VGA 메모리로 출력하는 함수
void gfx_present(void)
{
    gfx_wait_vsync();
	
    uint32_t* dst = (uint32_t*)vga;
    uint32_t* src = (uint32_t*)backbuffer;
	
    const int dwords = (WIDTH * HEIGHT) / 4;
    for (int i = 0; i < dwords; ++i)
    {
        dst[i] = src[i];
    }
}

// VSYNC 동기화 함수
void gfx_wait_vsync(void)
{
    while (inb(0x3DA) & 0x08) { }
    while (!(inb(0x3DA) & 0x08)) { }
}
```

기존 `gfx_present_with_cursor` 함수에서 **커서를 오버레이하는 부분을 분리하여 커서 모듈의 `cursor_render` 함수로 이전**하고, 화면 프레임 백 버퍼의 내용을 VGA에 출력하는 부분만 남겨두어 `gfx_present` 함수로 조정하였습니다.

**수직 동기화(V-Sync)**를 위하여 **VGA Input Status Register 1(**`0x3DA`** 포트)의 **`bit3`**(Vertical Retrace) 상태를 폴링하는 **`gfx_wait_vsync` **함수를 구현**하였습니다.


**cusor.h**

```C
...

// 현재 draw target에 커서를 렌더링하는 함수
void cursor_render(void);
```

현재 프레임 백 버퍼에 커서를 렌더링하는 함수 `cursor_render`를 선언하였습니다.


**cursor.c**

```C
...

void cursor_render(void)
{
    for (int y = 0; y < CURSOR_H; ++y)
    {
        for (int x = 0; x < CURSOR_W; ++x)
        {
            gfx_putpixel(cx + x, cy + y, COLOR_LIGHT_MAGENTA);
        }
    }
}

...
```

설정된 커서의 범위(`8x8`) 만큼을 밝은 심홍색으로 **화면 백 버퍼에 오버레이**하도록 함수 `cursor_render`를 구현하였습니다.


**kernel.c**

```C
...

void kernel_main(void)
{   
	wm_composite();
    cursor_render();
    gfx_present();
    
    ...
    
    wm_composite();
    cursor_render();
    gfx_present();
}
```

기존 `gfx_present_with_cursor()` 함수를 사용한 구간을 다음 단계로 변경하였습니다.

1. `wm_composite()` 함수를 통해 **윈도우 백 버퍼 내용을 화면 백 버퍼에 렌더링**합니다.
2. `cursor_render()` 함수를 통해 **커서를 화면 백 버퍼에 오버레이하여 렌더링**합니다.
3. `gfx_present()` 함수를 통해 **화면 백 버퍼 내용을 VGA 프레임버퍼에 출력**합니다.

# 이슈 및 버그

## 증상

 - 화면 출력 시 가끔 프레임 상/하단이 분리되어 보이는 **화면 티어링 현상**이 발생

## 재현

 - `gfx_wait_vsync()` 함수를 제거

## 원인(추정)

 - 화면이 스캔되는 도중 VGA 메모리가 갱신되어, 동일한 프레임 내에 **이전 프레임과 새로운 프레임이 혼재된 화면 티어링 현상**이 발생한 것으로 추정됨.

## 해결 및 우회 방법

 - `gfx_present()` 함수를 통해 화면 백 버퍼 내용을 VGA 프레임버퍼에 출력하기 전, `gfx_wait_vsync()` 함수를 호출해 **수직 리트레이스 구간에 맞추어 VGA 메모리를 갱신**하도록 수정하여 해결하였음.
