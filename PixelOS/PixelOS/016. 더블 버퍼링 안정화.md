# 변경 사항

## 코드

**graphics.h**

```C
...

// 백 버퍼 포인터 반환 함수
uint8_t* gfx_get_backbuffer(void);
```

**graphics.c**

```C
...

// 백 버퍼 포인터 반환 함수
uint8_t* gfx_get_backbuffer(void)
{
    return backbuffer;
}
```

화면 프레임 백 버퍼를 반환하는 함수 `gfx_get_backbuffer()`를 선언하였습니다.


**window.c**

```C
void wm_composite(void)
{
    ...
	
    cursor_composite(gfx_get_backbuffer());

```

윈도우 합성 함수(`wm_composite()`)의 **마지막 단계에서 커서를 백 버퍼에 함께 합성하도록 렌더링 흐름을 변경**했습니다.  

- **커서가 항상 최상단에 표시**되며, 윈도우에 의해 가려질 가능성을 방지하였습니다.
- 커서 렌더링을 최종 합성 단계로 일원화하여 **화면 출력의 일관성**을 높였습니다.


**kernel.c**

```C
...

void kernel_main(void)
{   
	wm_composite();
    gfx_present();
    
    ...
    
    wm_composite();
    gfx_present();
}
```

**커서 합성 단계가 윈도우 합성 함수에 추가됨**에 따라, **커널 코드에서 커서를 합성하는 단계를 제거**하였습니다.


## **리팩토링**

**cursor.h**

```C
...

typedef struct {
    int x;
    int y;
    bool visible;
} Cursor;

...

// 커서 합성 함수
void cursor_composite(uint8_t* backbuffer);
```

커서 관련 정보를 저장하는 구조체 `Cursor`를 생성하였습니다.

**화면 백 버퍼에 커서 모양을 합성하는 함수** `cursor_composite`를 선언하였습니다.
(기존의 `cursor_render()` 함수를 대체합니다.) 

- 기존 함수에서는 커서를 백 버퍼에 렌더링하기 위해 `gfx_putpixel()` **함수를 호출**하였으나, 대체된 함수에서는 **함수 호출 오버레이를 줄이기 위하여 직접 백 버퍼에 렌더링하도록 수정**되었습니다.

- 함수명을 변경하여 **함수의 의도가 커서의 오버레이 합성임**을 명확하게 하였습니다.


**cursor.c**

```C
...

#define CURSOR_COLOR COLOR_LIGHT_MAGENTA

static Cursor g_cursor;

...

void cursor_composite(uint8_t* backbuffer)
{
    if (!g_cursor.visible)
        return;
		
    for (int y = 0; y < CURSOR_H; ++y)
    {
        int sy = g_cursor.y + y;
        if (sy < 0 || sy >= HEIGHT) continue;
		
        for (int x = 0; x < CURSOR_W; ++x)
        {
            int sx = g_cursor.x + x;
            if (sx < 0 || sx >= WIDTH) continue;
			
            backbuffer[sy * WIDTH + sx] = CURSOR_COLOR;
        }
    }
}
```

`cursor_composite` 함수에서는 현재 커서의 `visible` 상태가 `true`인 경우 화면 백 버퍼에 `8x8` 크기의 커서를 합성하는 작업을 수행하며, 자체적으로 클리핑 작업을 수행합니다.
